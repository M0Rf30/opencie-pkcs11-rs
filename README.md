# opencie-pkcs11

Rust bindings for [libopencie-pkcs11](https://github.com/M0Rf30/opencie-pkcs11) — the PKCS#11 interface for the Italian CIE (Carta d'Identità Elettronica).

The crate exposes:

- **PKCS#11 standard interface** — the full Cryptoki C API generated by `bindgen`, plus safe Rust wrappers for the common session/sign/encrypt/decrypt operations.
- **CIE extensions** — `cie_enable`, `cie_disable`, PIN management, PDF signing, signature verification, reader watching, and the DigestInfo helper, all wrapped in idiomatic Rust.

The standalone `cie_sign_*` C++ Sign SDK is no longer exported from the C library and has been removed from these bindings (see [Changelog](#changelog)). PDF signing now goes through `cie::sign`.

## Installation

```toml
[dependencies]
opencie-pkcs11 = "0.2"
```

## Requirements

### Runtime

- `libopencie-pkcs11` shared library installed on the target system. Install from the upstream [releases](https://github.com/M0Rf30/opencie-pkcs11/releases) or build from source with [Meson](https://github.com/M0Rf30/opencie-pkcs11#building).

### Build

- A working `clang` toolchain (required by `bindgen`).
  - Debian/Ubuntu: `apt-get install libclang-dev`
  - Fedora/RHEL: `dnf install clang-devel`
  - macOS: `xcode-select --install`
  - Windows: install LLVM from <https://releases.llvm.org/>
- A checkout of the upstream C library so `build.rs` can run `bindgen` against its public headers.
  - Default location: a sibling directory `../opencie-pkcs11`.
  - Override with `OPENCIE_PKCS11_DIR=/path/to/opencie-pkcs11`.

`build.rs` does not vendor the C headers; only public headers under `include/opencie/` and `shared/src/pkcs11/pkcs11.h` are consumed.

## Usage

### PKCS#11 standard operations

```rust
use opencie_pkcs11::{ffi, pkcs11};

fn main() -> Result<(), pkcs11::Error> {
    pkcs11::initialize()?;

    let slots = pkcs11::get_slot_list(true)?;
    let session = pkcs11::open_session(
        slots[0],
        ffi::CKF_SERIAL_SESSION | ffi::CKF_RW_SESSION,
    )?;
    pkcs11::login(session, ffi::CKU_USER, "12345678")?;

    // ... cryptographic operations ...

    pkcs11::logout(session)?;
    pkcs11::close_session(session)?;
    pkcs11::finalize()
}
```

### CIE enrolment, PDF signing, verification

```rust
use opencie_pkcs11::cie;

fn main() -> Result<(), opencie_pkcs11::pkcs11::Error> {
    let pan = "1234567890123";
    let pin = "12345678";

    if !cie::is_enabled(pan) {
        cie::enable(pan, pin)?;
    }

    // PNG bytes drawn into the signature widget; pass None to omit the stamp.
    let stamp: Option<&[u8]> = None;

    cie::sign(
        "input.pdf",
        "PDF",
        pin,
        pan,
        0,                    // page (0-based)
        100.0, 100.0,         // x, y in points
        200.0, 50.0,          // width, height in points
        stamp,
        "signed.pdf",
    )?;

    let count = cie::verify("signed.pdf", None, 0, None)?;
    for i in 0..count {
        let info = cie::get_verify_info(i)?;
        println!("{}. {} {} — valid: {}", i, info.name, info.surname, info.is_sign_valid);
    }

    cie::extract_p7m("document.pdf.p7m", "document.pdf")?;
    Ok(())
}
```

### Reader management

```rust
use opencie_pkcs11::cie;

let initial = cie::reader_count();
println!("readers connected: {}", initial);
println!("first reader: {:?}", cie::reader_name());

// Block until the reader count changes (e.g. card inserted/removed).
let new_count = cie::reader_watch(initial);
println!("reader count is now {}", new_count);
```

### DigestInfo helper for raw RSA signing

```rust
use opencie_pkcs11::cie;

const NID_SHA256: i32 = 672;

let digest = [0u8; 32]; // 32-byte SHA-256 output
let der = cie::make_digest_info(NID_SHA256, &digest)?;
// Feed `der` into a raw RSA signing operation.
# Ok::<(), opencie_pkcs11::pkcs11::Error>(())
```

## Modules

- **`ffi`** — raw PKCS#11 bindings (generated by `bindgen`).
- **`cie_ffi`** — raw CIE extension bindings (generated by `bindgen`).
- **`pkcs11`** — safe wrappers around the standard PKCS#11 surface.
- **`cie`** — safe wrappers around the CIE extensions.

## Error handling

All fallible operations return `Result<T, pkcs11::Error>`, where `Error` wraps a `CK_RV` value:

```rust
use opencie_pkcs11::pkcs11::{self, Error};

match pkcs11::initialize() {
    Ok(()) => println!("initialized"),
    Err(Error(code)) => eprintln!("PKCS#11 error 0x{:08x}", code),
}
```

`cie::verify` and `cie::get_sign_count` return small positive counts; values at or above `0x1000` are interpreted as `CK_RV` error codes.

## Callback support

Progress, completion, and sign-completion callbacks defined in `cie_ext.h` are not yet exposed; the safe wrappers pass `NULL`. The raw `cie_ffi` module provides the typedefs for callers that need them today.

## Changelog

### 0.2.0

- Updated `cie::sign` to match the current `cie_ext.h`: the trailing `image_path: Option<&str>` parameter is replaced with `image_data: Option<&[u8]>`. **Breaking.**
- Removed the dead `sign` module and `sign_ffi` bindings: the C++ Sign SDK (`cie_sign_*` symbols) is no longer exported from `libopencie-pkcs11`.
- Added wrappers for `cie_get_verify_info`, `cie_reader_count`, `cie_reader_watch`, `cie_reader_name`, and `make_digest_info`.
- `build.rs` honours `OPENCIE_PKCS11_DIR` for non-default checkout locations.
